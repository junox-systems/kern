# Kern UI/UX Philosophy v0.1

The interface exists to disappear.

The user should spend their attention on their commitments, not on Kern.

Every interaction should reduce cognitive load.

# Principles

Attention over Information

Most productivity apps ask:

How can we show more?

Kern asks:

How can we show less while preserving confidence?

This changes everything.

⸻

## Principle 1 — Progressive Disclosure

Never show everything.

Reveal only what becomes useful.

Example:

Write proposal
Due tomorrow

Tap.

Why?
• Work allocation below target
• Fits current block
• Due tomorrow

Tap again.

Dependencies
History
Notes

Three levels.

Not one overwhelming screen.

⸻

## Principle 2 — One Decision per Screen

Every screen should answer one question.

Home

What deserves attention now?

Category

What's happening here?

Commitment

What is true?

Review

What changed?

Nothing else.

⸻

## Principle 3 — Calm Defaults

Nothing should blink.

Nothing should pulse.

Nothing should animate for attention.

Animations communicate state changes.

Never importance.

⸻

## Principle 4 — Reality First

Never punish reality.

If the operator skips something:

Skipped.
Recomputing...

Not

⚠️ Missed

Reality always wins.

⸻

## Principle 5 — Explain, Don’t Persuade

Every recommendation should answer:

Why this?

Not

Trust me.

Transparency builds trust.

⸻

## Principle 6 — Reduction before Organization

Whenever something feels crowded:

Don’t add tabs.

Don’t add filters.

Don’t add groups.

Remove.

⸻

## Principle 7 — Calm Hierarchy

Every page should have:

Primary
↓
Secondary
↓
Metadata

Nothing else.

Information density should come from proximity.

Not decoration.

⸻

## Principle 8 — Stable Layout

The interface should rarely move.

Things should live in consistent places.

ADHD benefits from recognition over search.

ASD benefits from spatial memory.

Muscle memory matters.

⸻

## Principle 9 — Quiet Typography

Fraunces:

Meaning.

Identity.

Reflection.

⸻

Atkinson Hyperlegible Next:

Everything else.

Reading should require almost zero effort.

⸻

## Principle 10 — Human Time

Avoid:

3 hours overdue

Prefer:

Due today

Avoid:

14 tasks remaining

Prefer:

One recommendation.
12 others can wait.

Language should reduce anxiety.

⸻

## Principle 11 — Recommendations are Temporary

Never make recommendations feel permanent.

They should look lightweight.

Almost ephemeral.

Because they’ll disappear after recompilation.

⸻

## Principle 12 — The Home Screen

The home screen should never feel like a dashboard.

Dashboards invite analysis.

Kern invites action.

I imagine something closer to:

Today
09:42
Deep Work
────────────
Write proposal
Why?
Work allocation
↓
Fits current block
↓
Due tomorrow
────────────
Later
• Exercise
• Review invoices
• Read paper

No charts.

No metrics.

No gamification.

⸻

## Principle 13 — Silence is Success

If nothing needs attention:

Say so.

You're caught up.
Go live your life.

Not

100%
🎉

⸻

## Principle 14 — Remove Navigation

Navigation is hidden decision-making.

Ask:

Can this be one tap fewer?

Every screen should have a reason to exist.

⸻

## Principle 15 — Software as Infrastructure

The operator should gradually stop noticing Kern.

Like:

* Git
* SQLite
* Raycast
* 1Password
* Obsidian

It should become trusted infrastructure.

Not a destination.

⸻

# The Mental Model

I think this is the biggest realization.

Most apps are designed like this:

```
User
↓
UI
↓
Data
```

Kern should be:

```
Reality
↓
Kernel
↓
UI
↓
Operator
```

The UI is not the product.

The Kernel is.

The UI simply makes the Kernel understandable.
